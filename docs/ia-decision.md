# 信息架构决策 · 展陈 / 集藏（layers-skills 梳理）

> 方法：Layers of Product Design 七层框架（问题空间→方案空间→表面）
> 原则：只记录**已做的决策**与**未决的开放问题**，不铺陈过程。
> 日期：2026-07-14

## 已确立的决策（decisions made）

**概念模型（L5）**
- 七顶凤冠 = 同一个 `Crown` 类型的七个实例（`destinations[]`），非七个不同对象。
  → OOUX 单一对象处理，避免 shapeshifter。
- 单冠的差异化属性（配色 `curtainColors`、字池 `charPool`、冠型 `art`、明暗 `theme`）均为该实例的字段，不是新类型。

**交互结构（L6）**
- 展陈页（单冠沉浸）与集藏页（七冠总览）为**同级视图切换**，由顶部 `SiteHeader` 的「展陈/集藏」切换，非嵌套路由（SPA、无 URL 路由，符合项目定位）。
- 集藏卡片点击 → 跳回展陈并定位该冠（`setIndex(i); setView("scene")`），集藏是**索引**而非平行画布。

**表面（L7）**
- 暗场沉浸主题，设计令牌集中在 `index.css :root`（dark-only，艺术项目有意为之）。
- 展陈页"其他冠引导"双路径：左右常驻胶囊（轻量顺手切 prev/next）+ 集藏入口（总览）。

## 未决的开放问题（open questions）

**Q1 — 集藏页的定位**
集藏页当前是"索引"（点卡片跳走）。是否需要在集藏页内**直接并排展示大图对比**（不跳走）？
→ 影响：概念模型（Crown 是否需要"对比视图"状态）+ 交互结构（是否引入第三视图）。
→ 当前 `GalleryViewPro` 仍是索引式（点击跳展陈），未解决。

**Q2 — 两条"引导"路径的权重** ✅ 已解（PR #31）
展陈页同时有左右胶囊和集藏按钮，功能部分重叠。哪条是主路径？
→ 决策：胶囊 = 顺手切（轻量 micro-nav，保持低调半透明描边）；集藏 = 总览（重）。
→ 落地：`SiteHeader` 导航按钮加 `rounded-md` + `hover:bg-white/5` + 当前态 `bg-white/[0.06]`，使"总览"入口视觉权重明显高于展陈页轻量胶囊。权重已区分。

**Q3 — 集藏页两版并存策略** ✅ 已解（PR #31）
`GalleryView`（默认网格）与 `GalleryViewPro`（`?pro=1` 高级画廊）并存。
→ 决策：默认网格版**吸收 Pro 精华**（去边框改留白分层、hover 光晕、冠图更大+投影），保留网格布局（七顶总览更实用）。Pro 版保留作对照/实验，推广期门面截图用升级后的默认版即可。
→ `?pro=1` 长期若不再需要应移除（见 F2）。

**Q4 — 深层状态 / 空态**
- 无"无内容"场景（七顶写死，不会空）——当前可接受。
- 冠图加载失败（CORS/404）时帘幕不渲染——已有 `public/_headers` 兜底，但缺**加载中/失败**的表面反馈（L7 缺口）。

## 风险标记（flagged bad decisions to watch）

- **F1**：展陈页"引导"双路径若视觉权重不区分，用户会困惑"这两个有什么区别"→ 见 Q2。
- **F2**：`?pro=1` 是推广期对比开关，若长期保留会变成"隐藏模式"债务；应在 Q3 决议后移除或转正。

## 下一步建议

1. 先解 Q2（区分两条引导路径的视觉权重）——低成本、高清晰。
2. 推广期用 Pro 版截图（已截 `docs/gallery-pro.png`）做门面，代码默认保持网格。
3. Q1（集藏内对比）若要做，单独开 PR，不阻塞当前推广。
